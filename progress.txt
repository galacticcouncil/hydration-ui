## Codebase Patterns
- Address book entries use `mode` (WalletMode enum) instead of `chainKey`; provider is optional on stored entries
- `getWalletModeByAddress(address)` derives mode; explicit `input.mode` overrides on add
- `AccountFilter` is whitelist-only: non-empty whitelist shows only those mode chips; empty/absent shows all five
- `AccountOption` renders ProviderLogo when provider is set, else chain icon via `getWalletModeIcon(mode)` (SubstrateH160 → Substrate icon)
- AddressBookModal filters by `address.mode` against whitelist (Default) or exact mode match; no PROVIDERS_BY_WALLET_MODE indirection
- Runnable self-checks live in `AddressBook.selfcheck.ts` and `AddressBook.store.selfcheck.ts` (esbuild + node, documented in file headers)
- Use fnm yarn at `~/.local/share/fnm/aliases/default/bin/yarn` when shell PATH lacks yarn
- Commit format for Ralph stories: `feat: [Story ID] - [Story Title]`
- Connect/account-select flows must pass explicit `whitelist={[WalletMode.Substrate, WalletMode.EVM]}` to AccountFilter to preserve pre-refactor chip set (Substrate + EVM only)
- Build `@galacticcouncil/web3-connect` before linting `apps/main` when address-book types change (`yarn workspace @galacticcouncil/web3-connect build`)
